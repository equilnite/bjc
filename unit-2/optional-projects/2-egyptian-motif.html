            <li> 
                In this project, you will repeatedly use rectangles, so first recall how to create a <code>rectangle</code> block. <br />
                <img class="indent" src="/img/2-complexity/EgyptianMotif_img/rectangleCode.png" height="275" alt="rectangle, width:(width) length:(length){pen down; repeat(2){move(width) steps; turn clockwise (90) degrees; move(length) steps; turn clockwise (90) degrees};pen up}" title="rectangle, width:(width) length:(length){pen down; repeat(2){move(width) steps; turn clockwise (90) degrees; move(length) steps; turn clockwise (90) degrees};pen up}"/>
                <div class="endnote"> Note the use of <code>pen down</code> and <code>pen up</code> blocks inside the rectangle definition. We don't normally do this but here this is done to simplify the more complex code coming ahead. This will ensure that once a rectangle is drawn, the pen is up and you don’t need to worry about unintended scribbles as your drawing sprite continues moving to execute the rest of the code.</div>

            </li>

                <img class="indent" src="/img/2-complexity/EgyptianMotif_img/pinwheelWithRectanglesCodeHeader.png"  height="100" alt="pinwheel with rectangles, branches:(number of branches) size:(size) backup:(backup) width:(width) length:(length)" title="pinwheel with rectangles, branches:(number of branches) size:(size) backup:(backup) width:(width) length:(length)"/><br />
 Its code will be identical to the <code>pinwheel</code> code except that it has the <code>rectangle</code> block inserted in between the two <code>move</code> blocks that created the pinwheel branches. <br />
                <img class="indent" src="/img/2-complexity/EgyptianMotif_img/rectangleInBetween.png"  height="175" alt="move(size) steps; rectangle, width:(width) length:(length); move(-1Xbackup) steps" title="move(size) steps; rectangle, width:(width) length:(length); move(-1Xbackup) steps"/><br />
